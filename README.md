# F1 Engine Shifter

iPhone and Android app version of the original Chrome typing extension. Keyboard input has been replaced with touch controls: start the engine, hold REV to raise RPM, brake to drop RPM, and use GEAR +/- paddles for manual shifting.

## Included

- Expo SDK 54 app for iOS and Android.
- Manual gear shifting from 1st to 8th gear.
- Hold-to-rev throttle behavior with RPM falloff and braking.
- RPM physics with throttle attack/release, gear-ratio RPM drops, redline bounce, and optional auto shift.
- Layered V10 sound pack generated locally as WAV assets.
- Shift-up and downshift one-shots.
- `expo-audio` playback with local assets, looping layers, playback-rate pitch changes, and volume crossfades.
- Mobile audio hardening: playback-rate clamping, throttled loop updates, shift one-shot pooling, master headroom, and AppState stop behavior.
- Driving feel polish: instant paddle shifts, shift cut/blip envelope, and redline limiter cut pulses.
- Sound realism pass: cycle-aligned stereo V10 engine loops, gear-dependent transmission whine, lift-off bark, V10 cylinder/bank texture, load-aware tone morphing, varied one-shot pitch, audio-side RPM/layer slew, shift RPM catch, overrun crackles, airbox scream, and throttle-on intake snap.
- Final mobile tuning: loop-player parameter caching, elapsed-time simulation ticks, and sound-pack peak/RMS reporting for safer real-device mixing.

## Real engine audio

I did not download or bundle YouTube audio. Most F1 onboard clips and broadcast recordings are copyrighted, so the app is built to accept real samples without shipping questionable assets. The included default pack is synthesized and royalty-free because it is generated by `scripts/generate-soundpack.mjs`.

For a more realistic release build, replace these local files with legally licensed loop samples:

- `assets/audio/v10_idle.wav`
- `assets/audio/v10_low.wav`
- `assets/audio/v10_mid.wav`
- `assets/audio/v10_high.wav`
- `assets/audio/shift_up.wav`
- `assets/audio/shift_down.wav`
- `assets/audio/gear_whine.wav`
- `assets/audio/lift_off.wav`
- `assets/audio/throttle_on.wav`
- `assets/audio/overrun_a.wav`
- `assets/audio/overrun_b.wav`
- `assets/audio/airbox_scream.wav`

Keep the same filenames and the app will use the new samples automatically.
Run `npm run generate:soundpack` after replacing samples to print a peak/RMS report and catch files that are too loud, too quiet, or too wide when collapsed to mono.

## Run

```sh
npm install
npm run generate:soundpack
npm start
```

Then open it in Expo Go, iOS Simulator, or Android Emulator.

## iPhone beginner path

1. Open `App Store` on the iPhone.
2. Search for `Expo Go`.
3. Install `Expo Go`.
4. On the iMac, open `Terminal` with `Spotlight`: press `Command + Space`, type `Terminal`, then press `Return`.
5. Move to this folder:

```sh
cd /Users/seung/Documents/Codex/2026-05-05/goal-2/f1-engine-shifter
```

6. Start Expo:

```sh
npm start
```

7. In the Terminal window, scan the QR code with the iPhone Camera app.
8. Tap the Expo Go notification that appears.

## Android beginner path
